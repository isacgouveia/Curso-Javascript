## Conceito de Função
É um **bloco de código** que realiza uma tarefa específica ou calcula um valor.

Ela é definida uma vez e pode ser chamada (ou invocada) várias vezes.

Funções ajudam a organizar o código, tornando-o reutilizável e mais fácil de entender.

<a id="page2"></a>

## Estrutura de uma função
Para declararmos uma função, utilizamos a palavra "function" seguido do nome desejado. O escopo da função é delimitado por chaves e tudo dentro delas é o que a função executará.

*Exemplo:*
```js
    // Estrutura da função
    function (){

    }

    // Exemplo
    function message(){
        console.log("Hello World!")
    }

    // Chamar a função
    message()
```

## Argumentos e Parâmetros
- **Argumentos** - É o valor que é passado para a função.
- **Parâmetros** - É a variável (escopo da função) que irá receber um valor em uma função.

*Exemplo:*
```js
    function message(PARÂMETRO){
        console.log("Olá", PARÂMETRO)
    }

    /* Exemplo 2*/
    function message(username){
        console.log("Olá", username)
    }

    message(ARGUMENTO)
    message("Isac")
    message("Geovana")

    /* Exemplo 2*/
    function sum(a,b){
        console.log(a + b)
    }

    sum(10,30)
    sum(5,4)

    /* Exemplo 3*/
    function joinText(text1,text2,text3){
        console.log(text1, text2, text3)
    }

    joinText("Isac", "Geovana", "Victor")
    joinText("Isac", "Victor") = Isac Victor UNDEFINED

    /* Substituindo o UNDEFINED por nada*/
    function joinText(text1 ="",text2 = "",text3 = ""){
        console.log(text1, text2, text3)
    }

    joinText("Isac", "Victor") = Isac Victor 


```
Observação:
- o parâmetro username foi criado dentro do escopo da função, então se precisarmos utilizar forá do escopo da função precisaremos declara-lá.
- O argumento será passado dentro da função message(ARGUMENTO).
- Caso não declarar todos os argumentos, o javascript irá substituir por UNDEFINED.

## Return
Quando se cria uma função, é possível executar instruções e cálculos dentro dela.
Pode-se utilizar a palavra-chave "return" para retornar um valor específico da função. Dessa forma, é possível utilizar o resultado fora da função, atribuino-o a uma variável ou exibindo-o diretamente com o console.log. O "return" permite que o valor calculado dentro da função seja acessado e utilizado em outras partes do código.

*Exemplo:*
```js
    function sum(a,b){
        let result = a + b

        return result
    }

    /* Primeira maneira */
    let response = sum(10,20)
    console.log(response)

    /* Segunda maneira*/
    console.log(sum(10,20))
```

## Escopo da Função
É possível utilizar a função antes mesmo dela ser declarada, por causa do Hoisting.

O que é hoisting? (conforme 1. Variáveis e Constantes.md)
- Hoisting (levantar iu içar) se refere ao comportamento do interpretador de mover as declarações de variáveis e FUNÇÕES para o topo do escopo em que foram definidas. antes mesmo da execução do código. possibilita usar uma variável ou função antes que ela esteja definida.

*Exemplo:*
```js
    message("Hello World!")

    function message(){
        console.log(message)
    }

    message("Isso é apenas um teste")
```

Podemos declarar função dentro de uma função.

*Exemplo:*
```js
    message("Hello World!")

    function message(){
        console.log(message)
        endLine()

        function endLine(){
            console.log("------")
        }
    }

    message("Isso é apenas um teste")

    // ERROR
    endLine()
```
Observação:
- Ao declarar o endLine() fora da função, retornara um erro pois, o escopo da função endLine() é somente da função message()

## Função Anonima
A função anonima é uma função que não tem nome, quando se utiliza? Quando criar uma função dentro de uma variável.

*Exemplo:*
```js
    // Exemplo de uma função anonima
    const showMessage1 = function(){
       return "Hello World!"
    }

    console.log(showMessage) // Retorna informando que se trata de uma função.

    console.log(showMessage()) // Retorna o Hello World!

    /* Podemos passar parâmetros*/
    const showMessage2 = function(name){
    return "Olá, " + name
    }

    console.log(showMessage2("Isac")) // Retorna Olá, Isac
```

## Arrow Function
Uma forma mais concisa de escrever funções anônimas.

*Exemplo:*
```js
    // Função anônima
    const showMessage = function (){
        console.log("Hello")
    }

    // Arrow Function
    const arrowFunction = () =>{
        console.log("Hello")
    }

    arroFunction() // Retorna Hello

    // Arrow Function + Parâmetros
    const arrowfunction2 = (name, email) =>{
        // Primeira forma
        console.log("Olá, ", name, " Seu e-mail é ", email)

        // Segunda forma (interpolação)
        console.log(`Olá ${name}, seu email é ${email}`)
    }

    arrowFunction2("Isac", "isac.gouveia@yitzhak.com.br")
```

## Callback Function
É uma função passada para outra função como argumento.

*Exemplo:*
```js
    function execute(taskName){
        console.log("Execuntando a tarefa: ", taskName)
    }

    execute("Fazendo o download")

    // Utilizando o callback
    function execute(taskname, callback){
        console.log("Executando a tarefa: ", taskname)

        callback()
    }

    function callback(){
        console.log("Tarefa finalizada")
    }

    execute("Fazendo o download", callback)

    // Callback com função anônima.
    execute("Fazendo o Upload", function (){
        console.log("Upload realizado com sucesso")
    })

    // Callback com arrow function
    execute("Iniciando o FTP", () =>{
        console.log("FTP iniciado com sucesso")
    })

    // Arrow function reduzido
    execute("Excluindo o arquivo...", () => console.log("Excluído com sucesso"))
```
Observação: Pode utilizar qualquer nome, mas para ficar claro utilizei o nome callback.
